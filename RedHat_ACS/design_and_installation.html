Kubernetes security can be implemented on four levels:

  1. In development, by integrating security testing early in continuous integration pipelines. (code)
  2. In distribution, by ensuring that the software supply chain is safe by verifying the integrity of workload components. (containers and registry)
  3. At deployment time, by validating workload attributes such as security parameters, and by using approved container images. (container, cluster)
  4. At runtime, by monitoring all elements of container runtime: the host operating system, network, storage, and container runtime; and ultimately restricting container activity to only sanctioned processes that are prevented from accessing unauthorized resources and from engaging in disallowed network communication.( container, cluster, cloud)
