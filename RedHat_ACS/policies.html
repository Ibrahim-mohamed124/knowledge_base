Default Security Policies

  • The default policy configuration is a baseline posture for detecting the most common risks in your environment.
  • When customizing default policies, it is a good practice to first clone the existing policy, apply any customizations to the new clone, and disable the original default policy.
  • Each policy falls into one of the categories that RHACS organizes by type and function. 

Enforcing a Deploy-time Policy

  • RHACS works with admission plug-ins in openshfit  to enforce policies before the their persistent in the ETCD and after the authentication and authorization layer. 
  • RHACS uses the ValidatingAdmissionWebhook controller to verify that the resource that is being provisioned complies with the specified security policies. RHACS creates a ValidatingWebhookConfiguration object, which contains multiple webhook rules.
  • When the Kubernetes or OpenShift API server receives a request that matches one of the webhook rules, the API server sends an AdmissionReview request to RHACS. Then, RHACS accepts or rejects the request, based on the configured security policies.
  • Complementing the admission controller, the Collector component analyzes and monitors container activity on the cluster nodes, including Linux kernel-level activity, container runtime activity, host network traffic, and file systems.

The Collector operates during run time, and the admission controller enforces policies during workload deployment.
