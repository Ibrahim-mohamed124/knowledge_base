security controls

  • An action or countermeasure to address security attacks (defensive measuers)
    • SQL injection attacks: Implement input validation and code reviews

Categories of security controls

  1. Implemented by people:
    1. managerial
    2. operational (on-prem)
  2. Technical controls:
    1. software
    2. hardware
    3. middleware (anti-malware programs and appliances)
  3. physical controls: (on-prem)

! HoneyPot: fake environment to attract attackers in order to discover more information about them

Types of security controls

  1. Preventive: Proactive action to stop before incident > polices
  2. Deterrent: Discourage users from doing somethings > warnings
  3. Detective: Monitor and identify incidents
  4. Corrective controls: Mitigate the effect of the attack > backups
  5. Compensating: Reduce the attack surface on a system that can not be replaced or upgraded

CIA triad

  1. Confidentiality: 
    • Preventing unauthorized people from getting information
    • Least privilege principle 
    • File encryption
  2. Integrity  
    • hashes
  3. Availability

Non-repudiation & AAA

  • Authentication, Authorization, Accounting(auditing or logging)
  • Non-repudiation: Attribution of an action to an individual for accountability
    • Digital signatures
    • PKI
    • Audit Logs
    • Time stamping 

Zero trust 

  • Unlike the legacy concept of internal and external to a Network ( Completely trusting the internals ), Zero Trust do not fully trust anyone
