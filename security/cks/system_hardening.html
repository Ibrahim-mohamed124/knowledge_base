Restricting kernel modules
  • To load a new module into the kernel
modprobe <modeule>
  • To list all the loaded modules
lsmod

 

  • To backlist a module put its name in /etc/modprobe.d/blacklist.conf file then reboot the node
vim  /etc/modprobe.d/blacklist.conf
blacklist sctp
blacklist dccp

  • To get the ID of a process  use pidof <process>
pidof etcd
  •  Then use the process id with strace
strace -p <pid>
  • To trace the system calls of containers use aquasec tracee, which uses ePBF programs to trace the system calls of the containers
  • To trace system calls of the host
podman run --name tracee --rm -it \
  --pid=host \
  --cgroupns=host \
  --privileged \
  -v /etc/os-release:/etc/os-release-host:ro \
  -v /boot:/boot:ro \
  aquasec/tracee:latest

--pid=host & --cgroupns=host: Allows Tracee to see processes running outside its own container (essential for monitoring the whole node).

  • Running Tracee with Container Filtering
podman run --name tracee --rm -it \
  --pid=host \
  --cgroupns=host \
  --privileged \
  -v /etc/os-release:/etc/os-release-host:ro \
  -v /boot:/boot:ro \
  aquasec/tracee:latest \
  --scope container=new
