Role Privileges Matrix

  • the built-in roles available within Portworx Backup User Management.
  • the Super Administrator, Infrastructure Administrator, Applications Administrator, and Applications User. 
  • RBAC resources are governed by user roles and permissions and non-RBAC resources, which are accessible regardless of role. 
  • The RBAC  resources:
    • Backup locations
    • Cloud accounts
    • Schedule policies
    • Backup Rules
    • Roles
    • Users and User Groups
  • Non-RBAC resources:
    • Clusters
    • Namespaces
    • Virtual machines
    • Backup schedules
    • Backups
    • Restores
  1. Infrastructure administrators, application administrator and application users can only view, edit, and delete the RBAC resources owned by them.
  2. Owners of Portworx Backup RBAC resources can share their resources with the intended users and groups regardless of the role of the users or groups.
  3. Portworx Backup roles can only view the non-owned owned RBAC resources shared with them by other roles.

Encryption Matrix

  • Portworx Backup allows you to enable two types of encryption, one for the backup data at rest in the destination and the other for the backups in transit.
  1. Server-side encryption (SSE-S3) only at AWS S3
  2. Transit data encryption

Backup Types by Driver Matrix

  • Portworx Backup uses different drivers to create backups
    • Portworx cloud snapshot: applies only if Portworx is your storage platform, utilizes Portworx driver to create a backup
    • Direct KDMP(Kubernetes Data Management Platform): utilizes KDMP driver to create a backup and holds good for cross-cloud backups and restores
    • CSI snapshot: uses storage providers CSI driver to create a snapshot
    • CSI snapshot + offload snapshot to backup location (KDMP driver): utilizes CSI driver to create a backup and KDMP driver to offload the backup to backup location (S3 or NFS)
