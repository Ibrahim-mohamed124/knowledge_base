To create a CA to self-sign other certificates

  1. Define the certificate authority name
CANAME="MY-ORG"

  2. create a private key for the CA

openssl genrsa -aes256 -out $CANAME.key 4096

 

  • Must add a passphrase to that key, this file contain both key pairs in the same file (public and private)
  • to see the public key use:
openssl rsa -in $CANAME.key -pubout

  3. create a root certificate for CA 

openssl req -x509 -new -nodes -key $CANAME.key -sha256 -days 1826 -out $CANAME.crt

 

  • provide the data about the organization
  • copy the root ca certificate to /usr/local/share/ca-certificates and update-ca-trust
  • define a MYCERT variable with the name of the certificate
MYCERT="anyname"

  4. create new private key for the new certificate 

 openssl req -new -nodes -out $MYCERT.csr -newkey rsa:4096 -keyout $MYCERT.key

  5. create a file to add the Subject Alternative Name (SAN) to SSL certificate

touch $MYCERT.ext

 

authorityKeyIdentifier = keyid,issuer
basicConstraints = CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
subjectAltName = @alt_names

[alt_names]
DNS.1 = localhost

   6. sign the CSR using the CA private key 

 openssl x509 -req -in $MYCERT.csr -CA  $CANAME.crt -CAkey  $CANAME.key -CAcreateserial -out promethues.crt -days 10 -sha256 -extfile $MYCERT.ext

  7. to renew an expired certificate with the same CA

  • Generate a new Key and CSR:
openssl req -new -nodes -newkey rsa:4096 -keyout portworx_new.key -out portworx_new.csr

 

  • Sign it with the CA
openssl x509 -req -in portworx_new.csr -CA $CANAME.crt -CAkey $CANAME.key -CAcreateserial -out portworx_new.crt -days 365 -sha256 -extfile anyname.ext

useful commands

  • To read a certificate
openssl x509 -in certificate.crt -text -noout
  • openssl x509 command to display the SAN attribute of a custom certificate
openssl x509 -ext subjectAltName -noout -in mycert.pem
