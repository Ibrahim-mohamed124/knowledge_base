• Arbiter
    • Arbiter nodes are a more cost-effective alternative to control plane nodes. They function similarly but run only the essential components required to maintain the etcd quorum and prevent a split-brain condition. Because they do not host the full control plane or any workloads, arbiter nodes can use less powerful hardware.

Installation steps:

  1. Generate the  discovery image from the hypered console.
    • Provide DNS that resolve public ip like api.openshift.com, which is the site that the discovery image  pull the coreos image from and join to the cluster after booting.
    • Static provide IPs from a subnet that has access to the internet to connect to the central console and the gateway should reach the internet.
    • create bond with two MAC addresses that are used for the server not for the ilo(out of band management)  (higher bandwidth 10Gb)
    • provide public ssh key that will be used to provision the OCP platform from the central console
    • Pull the iso image (minimal or full)
  2. Mount the iso image  through the ilo interface for each node after changing the boot order to boot from the CD
