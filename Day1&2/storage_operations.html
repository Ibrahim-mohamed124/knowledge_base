  • types of usable storage:

In descending order of preference, the preferred storage technology for the image registry is object storage, followed by block storage. Red Hat does not recommend using file storage for the image registry in production environments.

  • The configs.imageregistry.operator.openshift.io resource controls the configuration and intended status of the container image registry.
    • The image registry operator automatically configures the storage for installer-provisioned infrastructure clusters that run on AWS, Azure, GCP, IBM, or OpenStack. For those infrastructures, the image registry operator sets the spec.storage.managementState parameter to the Managed value and attempts to apply the default configuration to the underlying storage. If you do not want the image registry operator to apply the default settings to the storage, then set the spec.storage.managementState parameter to the Unmanaged value.
    • For other infrastructures, the image registry operator sets the spec.storage.managementState parameter to the Removed value. After you install the image registry operator, edit the spec.storage.managementState parameter and set it to the Managed value. Then, manually configure the storage for the image registry.
oc get objectbucketclaim \
  noobaa-registry -n openshift-storage \
  -o jsonpath='{.spec.bucketName}{"\n"}'

oc get route/s3 -n openshift-storage
oc edit configs.imageregistry.operator.openshift.io cluster
apiVersion: imageregistry.operator.openshift.io/v1
kind: Config
metadata:
  name: cluster
spec:

  replicas: 1
  storage:
    managementState: Managed
    pvc: null
    s3:  2
      bucket: noobaa-registry-02367470-0c7c-4c50-a640-11963ca54392  3
      region: us-east-1  4
      regionEndpoint: https://s3-openshift-storage.apps.ocp4.example.com  5

 

  • The image registry operator also creates the image-registry-private-configuration-user secret in the openshift-image-registry namespace. You use this secret to configure storage access and management. The credentials in this secret override the default credentials that the image registry operator uses.
oc create secret generic \
  image-registry-private-configuration-user \
  --from-literal=REGISTRY_STORAGE_S3_ACCESSKEY=myaccesskey \
  --from-literal=REGISTRY_STORAGE_S3_SECRETKEY=mysecretkey \
  --namespace openshift-image-registry
