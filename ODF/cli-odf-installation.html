• The installation depends on the deployment mode, but generally these steps are followed with minor modifications for each deployment type: 
    1. Prepare the storage nodes
    2. Install the local storage operator if a DAS  is used to provide the storage  backend 
    3. Create the volume set CR of the local storage operator after scanning for available disks
    4. Install the ODF operator
    5. Create a storage system using the created volume set
Prepare the storage nodes
  • Create  infra nodes to optimize the subscription cost if possible.
oc label nodes  <node_name> node-role.kubernetes.io/infra=
  • Add taints to the nodes so that application workloads does not get scheduled on it
oc adm taint nodes -l node-role.kubernetes.io/infra node-role.kubernetes.io/infra=reserved:NoSchedule node-role.kubernetes.io/infra=reserved:NoExecute
  • To use some of the infra nodes for storage only add this label to them:
oc label nodes   -l node-role.kubernetes.io/infra=   cluster.ocs.openshift.io/openshift-storage=
