• The installation depends on the deployment mode, but generally these steps are followed with minor modifications for each deployment type:
    1. Prepare the storage nodes
    2. Install the local storage operator if a DAS  is used to provide the storage  backend
    3. Create the volume set CR of the local storage operator after scanning for available disks
    4. Install the ODF operator
    5. Create a storage system using the created volume set
Prepare the storage nodes
  • Create  infra nodes to optimize the subscription cost if possible.
oc label nodes  <node_name> node-role.kubernetes.io/infra=

 

  • Add taints to the nodes so that application workloads does not get scheduled on it
oc adm taint nodes -l node-role.kubernetes.io/infra node-role.kubernetes.io/infra=reserved:NoSchedule node-role.kubernetes.io/infra=reserved:NoExecute

 

  • To use some of the infra nodes for storage only add this label to them:
oc label nodes   -l node-role.kubernetes.io/infra=   cluster.ocs.openshift.io/openshift-storage=

 

Install the local storage operator
  • Create the openshift-local-storage project
oc adm new-project openshift-local-storage

Projects starting with openshift- and kube- are default projects. These projects host cluster components that run as pods and other infrastructure components. As such, OpenShift Container Platform does not allow you to create projects starting with openshift- or kube- using the oc new-project command. Cluster administrators can create these projects using the oc adm new-project command.

  • Create the operator group and subscription
---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: openshift-local-storage
  namespace: openshift-local-storage
spec:
  targetNamespaces:
    - openshift-local-storage
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: local-storage-operator
  namespace: openshift-local-storage
spec:
  channel: stable
  installPlanApproval: Automatic
  name: local-storage-operator
  source: do370-catalog-cs
  sourceNamespace: openshift-marketplace
  • Wait till the operator is ready
watch oc get -n openshift-local-storage operatorgroups,subscriptions,clusterserviceversions
Create the volume set
  • Scan  for usable disks in the storage nodes by using the LocalVolumeDiscovery CR, which takes nodes selector and tolerations data
---
apiVersion: local.storage.openshift.io/v1alpha1
kind: LocalVolumeDiscovery
metadata:
  name: auto-discover-devices
  namespace: openshift-local-storage
spec:
  nodeSelector:
    nodeSelectorTerms:
    - matchExpressions:
      - key: cluster.ocs.openshift.io/openshift-storage
        operator: Exists
  tolerations:
  - key: node-role.kubernetes.io/infra
    value: reserved
    effect: NoSchedule
  - key: node-role.kubernetes.io/infra
    value: reserved
    effect: NoExecute
  • The results of the pervious  CR are stored in a CR called localvolumediscoveryresults
watch oc get localvolumediscoveryresults   -n openshift-local-storage
NAME                        AGE
discovery-result-worker01   4s
discovery-result-worker02   5s
discovery-result-worker03   6s
oc get localvolumediscoveryresults  -o custom-columns='NAME:metadata.name,PATH:status.discoveredDevices[*].path'   -n openshift-local-storage
  • Create the volume set, and determine the number of disk to be used in it or the minimum size of the disks:
---
apiVersion: local.storage.openshift.io/v1alpha1
kind: LocalVolumeSet
metadata:
  name: lso-volumeset
  namespace: openshift-local-storage
spec:
  deviceInclusionSpec:
    deviceTypes:
    - disk
    - part
    minSize: 0Ti
  nodeSelector:
    nodeSelectorTerms:
    - matchExpressions:
      - key: cluster.ocs.openshift.io/openshift-storage
        operator: Exists
  tolerations:
  - key: node-role.kubernetes.io/infra
    value: reserved
    effect: NoSchedule
  - key: node-role.kubernetes.io/infra
    value: reserved
    effect: NoExecute
  storageClassName: lso-volumeset
  volumeMode: Block
  maxDeviceCount: 1
  • Wait till the localvolumeset is ready
 watch oc get localvolumesets/lso-volumeset   -o custom-columns='"NAME:metadata.name,STORAGECLASS:spec.storageClassName,PROVISIONED:status.totalProvisionedDeviceCount,VOLUME_MODE:spec.volumeMode"' -n openshift-local-storage
Install the ODF operator
  • Create the openshift-storage namespace.
oc adm new-project openshift-storage
  • Label the openshift-storage namespace to use the available cluster monitoring solution.
oc label namespace/openshift-storage   openshift.io/cluster-monitoring=true
  • Create the operator group and subs:
---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: openshift-storage
  namespace: openshift-storage
spec:
  targetNamespaces:
    - openshift-storage
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: odf-operator
  namespace: openshift-storage
spec:
  channel: "stable-4.16"
  installPlanApproval: Automatic
  name: odf-operator
  source: do370-catalog-cs
  sourceNamespace: openshift-marketplace
  • Wait till the operator is ready
watch oc get -n openshift-storage   operatorgroups,subscriptions,clusterserviceversions

  • Enable the OpenShift Data Foundation Console plug-in by editing the console.operator cluster resource and adding the odf-console to the spec.plugins section
 oc patch console.operator cluster  -n openshift-storage --type json -p  "[{'op': 'add', 'path': '/spec/plugins', 'value': ['monitoring-plugin', 'odf-console']}]"
Create the storage system
  • create the storageCluster CR
---
apiVersion: ocs.openshift.io/v1
kind: StorageCluster
metadata:
  name: ocs-storagecluster
  namespace: openshift-storage
spec:
  monDataDirHostPath: /var/lib/rook
  storageDeviceSets:
  - count: 1
    dataPVCTemplate:
      spec:
        accessModes:
        - ReadWriteOnce
        resources:
          requests:
            storage: "1"
        storageClassName: lso-volumeset
        volumeMode: Block
    name: ocs-deviceset-lso-volumeset
    replica:  3
  flexibleScaling: false
  placement:
    all:
      tolerations:
      - key: node-role.kubernetes.io/infra
        value: reserved
        effect: NoSchedule
      - key: node-role.kubernetes.io/infra
        value: reserved
        effect: NoExecute
    mds:
      tolerations:
      - key: node-role.kubernetes.io/infra

        value: reserved
