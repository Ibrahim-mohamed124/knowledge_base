Resource Quotas for Storage Resources

  • OpenShift provides two types of quotas to manage the resource consumption, one for single projects and another for multiple projects, To create a resource quota for multiple projects, define a shared cluster resource quota to restrict resource usage across more than one namespace.
  • By using the storage-class-name.storageclass.storage.k8s.io prefixes, you can specify the resource quota for a storage class. With this approach, specialized storage classes, such as those with faster throughput disks or larger storage pools, can have stricter constraints.
  • You can set resource quotas across projects to enforce a baseline storage allocation for the applications in each project. 
apiVersion: quota.openshift.io/v1
kind: ClusterResourceQuota
metadata:
  name: for-user
spec:
  quota: 1
    hard:
      persistentvolumeclaims: "2"
  selector:
    annotations: 2
      openshift.io/requester: username
    labels: null 3
oc describe appliedclusterresourcequotas
