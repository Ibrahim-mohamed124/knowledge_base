• OpenShift cannot directly use volume snapshots. Thus, you must clone a volume snapshot to a PV to be able to use it. For RBD volumes, OpenShift Data Foundation internally uses layered clone and copy-on-write techniques to reduce the storage usage. For CephFS volumes, OpenShift Data Foundation copies all the data from the source volume, and the volume clone takes the same space as the source volume.
  • In cloning, OpenShift creates a volume snapshot of the volume and a clone of that snapshot. However, the temporary snapshot is not exposed on the OpenShift side.
  • A volume snapshot is the state of a PV in the cluster at a particular time. Volume snapshots help with storage efficiency because you do not need to make a full PV copy each time.
  • The CSI snapshot controller handles the binding of the VolumeSnapshot CRD with a VolumeSnapshotContent CRD, similarly to the binding of a PVC to a PV. The VolumeSnapshotContent CRD is a cluster resource that points to a real snapshot in the storage back end.
  • Volume snapshots are namespaced resources. By default, the deletion policy for the volume snapshot classes is set to Delete. Thus, if you delete the volume snapshot object or the namespace, then the volume snapshot content is deleted. To preserve the volume snapshot content if you delete the volume snapshot object, use the following command to change the deletion policy for the volume snapshot class to Retain:
oc patch \
  volumesnapshotclass/ocs-storagecluster-cephfsplugin-snapclass \
  --type merge -p '{"deletionPolicy":"Retain"}'

apiVersion: snapshot.storage.k8s.io/v1
kind: VolumeSnapshot
metadata:
  name: postgresql-snapshot 1
spec:
  volumeSnapshotClassName: ocs-storagecluster-rbdplugin-snapclass 2
  source:
    persistentVolumeClaimName: postgresql-data 3

 

Create a PVC from a Volume Snapshot
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata: 1
  name: postgresql-data-restore
  namespace: backup-volume
  labels:
    app: postgresql-data-snapshot
spec:
  accessModes: 2
  - ReadWriteOnce
  storageClassName: ocs-storagecluster-ceph-rbd 3
  dataSource: 4
    apiGroup: snapshot.storage.k8s.io
    kind: VolumeSnapshot
    name: postgresql-snapshot
  resources:
    requests:
      storage: 1Gi

Volume Cloning

  • A volume clone is a duplicate of an existing storage volume. You create a clone of a volume to make a point-in-time copy of the data. The back-end device creates an exact duplicate of the specified volume instead of an empty volume.
  • You can create a clone of a PVC by referencing the source PVC in the dataSource parameter as follows:
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata: 1
  name: postgresql-clone
  namespace: backup-volume
  labels:
    app: postgresql-data-clone
spec:
  accessModes: 2
    - ReadWriteOnce
  storageClassName: ocs-storagecluster-ceph-rbd 3
  dataSource: 4
    kind: PersistentVolumeClaim
    name: postgresql-data
  resources:
    requests:
      storage: 1Gi
