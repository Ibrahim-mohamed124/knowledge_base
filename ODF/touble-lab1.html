• View the health status of the Ceph cluster by using the oc get command.
oc get cephcluster

 

  • View the cluster health details by using the oc describe command.
oc describe cephcluster | grep "^Status" -A15

Note

Ceph periodically checks the integrity of your data by scrubbing placement groups. A scrub error indicates that Ceph detects a consistency error in your data.

  • View the Ceph MON logs by using the oc logs command with the app=rook-ceph-mon label.
oc logs -l app=rook-ceph-mon --tail=-1 --since=1h \
  | grep ERR
  • View the Ceph OSD logs by specifying the app=rook-ceph-osd label for the oc logs command. Filter the log with the ERR flag.
oc logs -l app=rook-ceph-osd --tail=-1 --since=1h \
  | grep ERR
  • Create a Must-Gather archive 
oc adm must-gather \
  --image=registry.ocp4.example.com:8443/odf4/odf-must-gather-rhel9:v4.16 \
  --dest-dir=mg
  • Obtain the cluster health with the ceph health command.
ceph  health detail
  • Use the rados list-inconsistent-obj command to get more information about the inconsistency. 
 rados list-inconsistent-obj x.0 | jq .inconsistents[].shards

One or more causes can lead to PG inconsistencies. A PG repair can be dangerous in some cases and might result in a complete data loss.

Always consult with Red Hat support before attempting a PG repair.
