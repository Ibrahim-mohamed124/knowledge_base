• Configure the s3cmd tool by using the .s3cfg configuration file in the home directory. Update the access_key and secret_key fields by using the retrieved values from the previous step.
(app-root) bash-4.4$ vi ~/.s3cfg
access_key = AWS_ACCESS_KEY_ID
secret_key = AWS_SECRET_ACCESS_KEY
host_base = rook-ceph-rgw-ocs-storagecluster-cephobjectstore.openshift-storage.svc
host_bucket = rook-ceph-rgw-ocs-storagecluster-cephobjectstore.openshift-storage.svc
use_https = True
ca_certs_file = /run/secrets/kubernetes.io/serviceaccount/service-ca.crt
  • List all the available S3 buckets.
(app-root) bash-4.4$ s3cmd ls s3://
2024-08-30 08:11  s3://object-bucket-3d937944-9ece-48e0-be45-a5ec4646a18a
  • List the objects that are stored in the S3 bucket.
(app-root) bash-4.4$ s3cmd du s3://${BUCKET_NAME}
   0       0 objects s3://object-bucket-3d937944-9ece-48e0-be45-a5ec4646a18a/
  • Upload and access the objects in the S3 bucket.

Create a sample file and upload it to the S3 bucket.

bash-4.4$ s3cmd put sample-file.txt s3://${BUCKET_NAME}/
WARNING: sample-file.txt: Owner groupname not known. Storing GID=1000750000 instead.
upload: 'sample-file.txt' -> 's3://object-bucket-3d937.../sample-file.txt'  [1 of 1]
 12 of 12   100% in    0s   188.84 B/s  done
  • Retrieve the file from the S3 bucket and view its contents.
(app-root) bash-4.4$ s3cmd get s3://${BUCKET_NAME}/sample-file.txt /tmp/
download: 's3://object-bucket-3d937.../sample-file.txt' -> '/tmp/sample-file.txt'  [1 of 1]
 12 of 12   100% in    0s   229.39 B/s  done(app-root) bash-4.4$ cat /tmp/sample-file.txt
Hello world
  • Update the sample file and upload it again to the S3 bucket.
bash-4.4$ s3cmd put sample-file.txt s3://${BUCKET_NAME}/
WARNING: sample-file.txt: Owner groupname not known. Storing GID=1000750000 instead.
upload: 'sample-file.txt' -> 's3://object-bucket-3d937.../sample-file.txt'  [1 of 1]
 16 of 16   100% in    0s   213.35 B/s  done
  • Duplicate an object in the S3 bucket.
(app-root) bash-4.4$ s3cmd cp s3://${BUCKET_NAME}/sample-file.txt \
  s3://${BUCKET_NAME}/second-file.txt
remote copy: 's3://object-bucket-3d937.../sample-file.txt' -> 's3://object-bucket-3d937.../second-file.txt'  [1 of 1]
  • Retrieve the duplicate file from the S3 bucket and view its contents.
(app-root) bash-4.4$ s3cmd get s3://${BUCKET_NAME}/second-file.txt /tmp/
download: 's3://object-bucket-3d937.../second-file.txt' -> '/tmp/second-file.txt'  [1 of 1]
 16 of 16   100% in    0s   330.57 B/s  done(app-root) bash-4.4$ cat /tmp/second-file.txt
updated content
  • List all the stored objects in the S3 bucket.
(app-root) bash-4.4$ s3cmd la s3://${BUCKET_NAME}
2024-08-30 ...    16    s3://object-bucket-3d937.../sample-file.txt
2024-08-30 ...    16    s3://object-bucket-3d937.../second-file.txt

Create a file and upload the file with a prefix. The prefix enables objects to be sorted hierarchically.

(app-root) bash-4.4$ echo "New world" > new-file.txt(app-root) bash-4.4$ s3cmd put new-file.txt s3://${BUCKET_NAME}/prefix/
WARNING: new-file.txt: Owner groupname not known. Storing GID=1000750000 instead.
upload: 'new-file.txt' -> 's3://object-bucket-3d937.../prefix/new-file.txt'  [1 of 1]
 10 of 10   100% in    0s   231.59 B/s  done
  • Change the prefix name.
(app-root) bash-4.4$ s3cmd mv s3://${BUCKET_NAME}/prefix/ \
  s3://${BUCKET_NAME}/my-s3-prefix/ --recursive
move: 's3://object-bucket-3d937.../prefix/sample-file.txt' -> 's3://object-bucket-3d937.../my-s3-prefix/sample-file.txt'  [1 of 1]
  • List all the stored objects in the S3 bucket, and verify that the new prefix is available.
(app-root) bash-4.4$ s3cmd la s3://${BUCKET_NAME}/
                  DIR   s3://object-bucket-3d937.../my-s3-prefix/
2024-08-30 ...    16    s3://object-bucket-3d937.../sample-file.txt
2024-08-30 ...    16    s3://object-bucket-3d937.../second-file.txt
  • Rename an object that is stored in the S3 bucket.
(app-root) bash-4.4$ s3cmd mv s3://${BUCKET_NAME}/second-file.txt \
  s3://${BUCKET_NAME}/other-file.txt
move: 's3://object-bucket-3d937.../second-file.txt' -> 's3://object-bucket-3d937.../other-file.txt'  [1 of 1]
  • Move an object inside a given prefix.
(app-root) bash-4.4$ s3cmd mv s3://${BUCKET_NAME}/sample-file.txt \
  s3://${BUCKET_NAME}/my-s3-prefix/
move: 's3://object-bucket-3d937.../sample-file.txt' -> 's3://object-bucket-3d937.../my-s3-prefix/sample-file.txt'  [1 of 1]
  • List all the stored objects in the S3 bucket.
(app-root) bash-4.4$ s3cmd ls s3://${BUCKET_NAME}/ -r
