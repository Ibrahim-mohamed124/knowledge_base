Instrumentation using promethues client library

  • To embed custom metrics into an application, import the promethues library for the programming language that is used, and use its functions to create metrics. For example:
pip install prometheus-client
  • Then use this library functions:
from flask  import Flask
from prometheus_client import Counter, start_http_server

# initialize the Counter function to create a counter metric
REQUESTS= Counter('metric_name', 'Metric describtion')


app = Flask(__name__)



@app.get("/cars")
def get_cars():
# to increament the counter every time this endpoint is called
    REQUESTS.inc()
    return ["Toyota"]

if __name__  == '__main__':
# to start http server to expose the metrics
    start_http_server(8001)
    app.run(port='5001')

 

  • Or you can use the framework logic (like flask) to expose the metrics on the same port of the application but in different endpoint by using the promethues client library function to expose the metrics in that endpoint. 
from flask  import Flask
from werkzeug.middleware.dispatcher import DispatcherMiddleware
from prometheus_client import Counter, make_wsgi_app # this function export metrics to the app server 


REQUESTS= Counter('http_requests_total', 'Total number of requests')


app = Flask(__name__)
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {'/metrics': make_wsgi_app()})



@app.get("/cars")
def get_cars():
    REQUESTS.inc()
    return ["Toyota"]

if __name__  == '__main__':
    app.run(port='5001')
