Instrumentation using promethues client library

  • To embed custom metrics into an application, import the promethues library for the programming language that is used, and use its functions to create metrics. For example:
pip install prometheus-client

 

  • Then use this library functions:
from flask  import Flask
from prometheus_client import Counter, start_http_server

# initialize the Counter function to create a counter metric
REQUESTS= Counter('metric_name', 'Metric describtion')


app = Flask(__name__)



@app.get("/cars")
def get_cars():
# to increament the counter every time this endpoint is called
    REQUESTS.inc()
    return ["Toyota"]

if __name__  == '__main__':
# to start http server to expose the metrics
    start_http_server(8001)
    app.run(port='5001')

 

  • Or you can use the framework logic (like flask) to expose the metrics on the same port of the application but in different endpoint by using the promethues client library function to expose the metrics in that endpoint.
from flask  import Flask
from werkzeug.middleware.dispatcher import DispatcherMiddleware
from prometheus_client import Counter, make_wsgi_app # this function export metrics to the app server 


REQUESTS= Counter('http_requests_total', 'Total number of requests')


app = Flask(__name__)
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {'/metrics': make_wsgi_app()})



@app.get("/cars")
def get_cars():
    REQUESTS.inc()
    return ["Toyota"]

if __name__  == '__main__':
    app.run(port='5001')
  • to add labels to metrics to be able to do operations on them while distinguishing the sub categories, use this
from flask  import Flask
from werkzeug.middleware.dispatcher import DispatcherMiddleware
from prometheus_client import Counter, make_wsgi_app


# define the labels names
REQUESTS= Counter('http_requests_total', 'Total number of requests', labelnames=['path', 'method'])


app = Flask(__name__)
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {'/metrics': make_wsgi_app()})



@app.get("/cars")
def get_cars():
# define the labels values
    REQUESTS.labels('/cars', 'get').inc()
    return ["Toyota"]


@app.get("/cycles")
def get_cycles():
    REQUESTS.labels('/cycles', 'get').inc()
    return ["Motorcycle"]


if __name__  == '__main__':
    app.run(port='5001')
  • To create histograms, import the Histogram function and implement the logic for the metric
from flask  import Flask, request
from werkzeug.middleware.dispatcher import DispatcherMiddleware
from prometheus_client import Counter, Histogram , make_wsgi_app
import time


REQUESTS = Counter('http_requests_total', 'Total number of requests', labelnames=['path', 'method'])
LATENCY = Histogram('request_latency_seconds', 'Request Latency', labelnames=['path', 'method'])

app = Flask(__name__)
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {'/metrics': make_wsgi_app()})

@app.before_request
def before_request():
    request.start_time  = time.time()
@app.after_request
# This is where the histogram gets its value - the observe method () -
def after_request(response):
    request_latency = time.time() - request.start_time
    LATENCY.labels(request.path, request.method).observe(request_latency)
    return response


@app.get("/cars")
def get_cars():
    REQUESTS.labels('/cars', 'get').inc()
    return ["Toyota"]


@app.get("/cycles")
def get_cycles():
    REQUESTS.labels('/cycles', 'get').inc()
    return ["Motorcycle"]


if __name__  == '__main__':
    app.run(port='5001')

 

  • To make a gauge metric, use the Gauge function and use the sub functions inc(), dec(), set() to implement the increase and decrease logic of the guage
from flask  import Flask, request
from werkzeug.middleware.dispatcher import DispatcherMiddleware
from prometheus_client import Counter, Histogram , make_wsgi_app, Gauge
import time


REQUESTS = Counter('http_requests_total', 'Total number of requests', labelnames=['path', 'method'])
LATENCY = Histogram('request_latency_seconds', 'Request Latency', labelnames=['path', 'method'])
IN_PROGRESS = Gauge('my_inprogress_requests', 'my guage', labelnames=['path', 'method'])
app = Flask(__name__)
app.wsgi_app = DispatcherMiddleware(app.wsgi_app, {'/metrics': make_wsgi_app()})

@app.before_request
def before_request():
    request.start_time  = time.time()
    IN_PROGRESS.labels(request.path, request.method).inc()
@app.after_request
def after_request(response):
    time.sleep(20)
    request_latency = time.time() - request.start_time
    LATENCY.labels(request.path, request.method).observe(request_latency)
    IN_PROGRESS.labels(request.path, request.method).dec()
    return response


@app.get("/cars")
def get_cars():
    REQUESTS.labels('/cars', 'get').inc()
    return ["Toyota"]


@app.get("/cycles")
def get_cycles():
    REQUESTS.labels('/cycles', 'get').inc()
    return ["Motorcycle"]


if __name__  == '__main__':
    app.run(port='5001')
  • metrics naming best practices:
    • the name should always follow the snake_case
    • the first word is for the package that will be monitored (http, postgres, node)
    • the second part should be the metric name
    • the last part should be the measuring unit (expect the counters)
