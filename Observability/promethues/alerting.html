Alert Manager:

  • Prometheus generates alerts based on conditions written in PromQL, then alert manager takes these alerts and routes them to a notification mechanism.
  • Alerting rules are used by promethues to write the conditions, on which Prometheus will generate alerts 
groups:
  - name: node
    interval: 15s
    rules:
     - record: ...
     - alert: LowMemory
       expr: node_memory_memFree_percent < 20
  • To tell promethues to evaluate the condition for some time before firing the alert use the for attribute 
groups:
  - name: node
    interval: 15s
    rules:
     - record: ...
     - alert: NodeDown
       expr: up{job="node"} == 0
       for: 10m
  • Alert states:
    • Inactive: the condition is false
    • Pending: the condition is true but waiting for the for period to end
    • Firing: condition is true
