Alert Manager:

  • Prometheus generates alerts based on conditions written in PromQL, then alert manager takes these alerts and routes them to a notification mechanism.
  • Alerting rules are used by promethues to write the conditions, on which Prometheus will generate alerts 
groups:
  - name: node
    interval: 15s
    rules:
     - record: ...
     - alert: LowMemory
       expr: node_memory_memFree_percent < 20
  • To tell promethues to evaluate the condition for some time before firing the alert use the for attribute 
groups:
  - name: node
    interval: 15s
    rules:
     - record: ...
     - alert: NodeDown
       expr: up{job="node"} == 0
       for: 10m
  • Alert states:
    • Inactive: the condition is false
    • Pending: the condition is true but waiting for the for period to end
    • Firing: condition is true
  • To add a label to an alert
groups:
  - name: node
    interval: 15s
    rules:
     - record: ...
     - alert: NodeDown
       expr: up{job="node"} == 0
       for: 10m
       labels:
         severity: high
       annotations:
         description: "only for describtion that are able to use go templating language"
  • To access the labels in annotations 
{{.Labels}} > to get access to alert labels
{{.Lables.instance}}  > to get acess to instance labels
{{.Value}} > to get the metric value when the alert is firing

Alert Manager architecture:

  • One alert manager can be used with many promethues servers
  • Dispatching: groups the alerts together
  • inhibition: suppress alerts if other alerts exist
  • silencing: silence alerts by silence rules
  • routing:  what alert get sent to who ?
  • notification engine: send alerts

Installation:

  • Get the binray
wget https://github.com/prometheus/alertmanager/releases/download/v0.31.1/alertmanager-0.31.1.linux-amd64.tar.gz
tar -xvaf alertmanager-0.31.1.linux-amd64.tar.gz
  • configure promethues to use alert manager:
global:
alerting:
 alertmanagers:
 - static_configs:
   - targets:
     - alertmanager_IP:9093
rule_files:
 - "rules.yml"
  • To check rules syntax
[root@k8s-master ~]# promtool check rules /etc/prometheus/rules.yml
