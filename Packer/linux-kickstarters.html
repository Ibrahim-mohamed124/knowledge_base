Linux Distribution Automation Methods

Key Differences

RPM-based (Kickstart)

  • Syntax: INI-style with sections (%packages, %post, etc.)
  • Flexibility: Very powerful scripting in %post and %pre
  • Example:

 

 

bash

install text
url --url=http://mirror.example.com/rhel9
lang en_US.UTF-8
keyboard us
%packages
@core
%end

Debian-based (Preseed)

  • Syntax: Key-value pairs (debconf format)
  • Complexity: Less intuitive, requires understanding debconf
  • Example:

 

 

bash

d-i debian-installer/locale string en_US
d-i keyboard-configuration/xkb-keymap select us
d-i netcfg/get_hostname string debian-host

Modern Ubuntu (Autoinstall)

  • Syntax: YAML-based (cloud-init)
  • Modern: More user-friendly than preseed
  • Example:

 

 

yaml

autoinstall:   version: 1   identity:     hostname: ubuntu-server
    username: ubuntu
    password: "$6$encrypted..."

SUSE (AutoYaST)

  • Syntax: XML-based
  • GUI Tool: Can generate configs via YaST
  • Example:

 

 

xml

<profile>   <install>     <image>false</image>   </install> </profile>
