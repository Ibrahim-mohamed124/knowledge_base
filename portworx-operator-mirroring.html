Mirroring the operator bundle to internal quay
  • First, create the ImageSetConfiguration 
kind: ImageSetConfiguration
apiVersion: mirror.openshift.io/v1alpha2
mirror:
  operators:
    - catalog: registry.redhat.io/redhat/certified-operator-index:v4.21
      packages:
        - name: portworx-certified
          channels:
            - name: stable
              minVersion: '2.12.0' # Replace with your target version
              maxVersion: '2.12.0'

 

  • Mirror the images
[root@mirror imageset]# oc-mirror -c imageset.yaml \
  docker://mirror.ocp4.plus.local:8443 \
  --v2 \
  --workspace file:///oc-mirror-workspace #The files will be created here\
  --dest-tls-verify=false \
  --authfile config.json # you can get this file by using podman login and copy the auth file \
  --remove-signatures \
  --retry-times 10 \
  --log-level debug

 

  • In the disconnected cluster add the custom portworx catalog, these files will be created automatically by oc-mirror command in the worspace
# 1. Update the image pull redirects (triggers node rollouts)
oc apply -f /oc-mirror-workspace/working-dir/cluster-resources/idms-oc-mirror.yaml

# 2. Add the Portworx catalog to OperatorHub
oc apply -f /oc-mirror-workspace/working-dir/cluster-resources/cs-certified-operator-index-v4-21.yaml

 

  • If not set, disable the default catalogs
oc patch operatorhub cluster --type='json' -p='[{"op": "add", "path": "/spec/disableAllDefaultSources", "value": true}]'
Preparing for portworx installation
  • On your internet-connected host, set an environment variable for the Kubernetes version that you are using:
KBVER=$(oc version | awk -F'[v+_-]' '/Kubernetes/ {print $2}')

 

  • Set an environment variable to the latest major version of Portworx:
PXVER=<portworx-version>

 

  • On an internet-connected host with the same architecture and OS version as the Kubernetes cluster nodes intended for Portworx installation, download the air-gapped installation bootstrap script for the specified Kubernetes and Portworx versions:
curl -o px-ag-install.sh -L "https://install.portworx.com/$PXVER/air-gapped?kbver=$KBVER"

 

  • Pull the container images required for the specified versions:
sh px-ag-install.sh pull

 

  • Authenticate to the internal registry
podman login -u -p https://mirror.ocp4.linux.plus:8443

 

  • Push the container images to your internal  cluster registry.
sh px-ag-install.sh push https://mirror.ocp4.linux.plus:8443

 

  • Create a secret for the Operator to use that contains the registry credentials.
oc -n portworx create secret docker-registry px-image-repository \
    --docker-server=mirror.ocp4.linux.plus:8443 \
    --docker-username=openshift\
    --docker-password=redhat123

Enabling default openshift image registry

Phase 1: Enable the Internal Registry

By default, the registry may be in a "Removed" state or lacks a public route.

1. Set Management State to Managed

Ensure the Image Registry Operator is actively managing the registry.

oc patch configs.imageregistry.operator.openshift.io/cluster --type=merge --patch '{"spec":{"managementState":"Managed"}}'

2. Configure Storage (The Fix for "Degraded" status)

The registry requires a backend to store image layers. On Bare Metal, using a PVC is the standard approach.

oc patch configs.imageregistry.operator.openshift.io/cluster --type=merge --patch '{"spec":{"storage":{"pvc":{"claim":""}}}}'

Note: This creates a PVC named image-registry-storage using the cluster's default StorageClass.

3. Expose the Default Route

Create a public URL so external tools (like Docker/Podman) can reach the registry.

Bash

 

oc patch configs.imageregistry.operator.openshift.io/cluster --patch '{"spec":{"defaultRoute":true}}' --type=merge

Phase 2: Prepare the Bastion Host

Since the registry uses self-signed certificates, the bastion host must be configured to trust the cluster's Certificate Authority (CA).

1. Trust the Cluster CA

Extract the CA from the cluster and add it to the system trust store.

 

# Extract CA
oc get secret -n openshift-ingress-operator router-ca -o jsonpath='{.data.tls\.crt}' | base64 --decode > cluster-ca.crt

# Add to System Trust (RHEL/CentOS)
sudo mv cluster-ca.crt /etc/pki/ca-trust/source/anchors/
sudo update-ca-trust extract

# Restart Docker to pick up changes
sudo systemctl restart docker

2. Configure Insecure Registry (Optional Backup)

  • If SSL issues persist, add the route to /etc/docker/daemon.json:
  • If podman will be used instead of docker, you can bypass the ssl by using the --verify-ssl false flag
{
  "insecure-registries": ["<your-registry-route-url>"] # for docker only
  }

Phase 3: Push the Image

1. Get the Registry URL

Retrieve the newly created route:

Bash

 

export REG_ROUTE=$(oc get route default-route -n openshift-image-registry -o jsonpath='{.spec.host}')
echo $REG_ROUTE

2. Grant Permissions

Grant your user permission to push images into your target project (e.g., postgres).

oc policy add-role-to-user system:image-writer $(oc whoami) -n postgres

3. Login, Tag, and Push

# Login using OpenShift Token
docker login -u $(oc whoami) -p $(oc whoami -t) $REG_ROUTE

# Tag the existing image
docker tag postgres:15-alpine $REG_ROUTE/test/postgres:15-alpine

# Push the image
docker push $REG_ROUTE/test/postgres:15-alpine

Phase 4: Verification

Verify that the image is now stored as an ImageStream in OpenShift.

oc get is -n test

Apply the posgres sts yaml file


---
# 1. Secret to store the database password
apiVersion: v1
kind: Secret
metadata:
  name: postgres-secret
type: Opaque
stringData:
  postgres-password: "StrongPassword123" # Change this!

---
# 2. Headless Service (Required for StatefulSet network identity)
apiVersion: v1
kind: Service
metadata:
  name: postgres-headless
  labels:
    app: postgres
spec:
  ports:
  - port: 5432
    name: postgres
  clusterIP: None
  selector:
    app: postgres

---
# 3. Public/Internal Service to expose the DB
apiVersion: v1
kind: Service
metadata:
  name: postgres-service
spec:
  type: ClusterIP # Change to LoadBalancer if on Cloud (AWS/GCP/Azure)
  ports:
  - port: 5432
    targetPort: 5432
  selector:
    app: postgres

---
# 4. The StatefulSet
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: postgres
spec:
  serviceName: "postgres-headless"
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: default-route-openshift-image-registry.apps.baremetal.plus.local/test/postgres:15-alpine
         # change the image to the internal image registry that has been configured perviously
         ports:
        - containerPort: 5432
          name: postgres
        env:
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: postgres-secret
              key: postgres-password
        - name: PGDATA
          value: /var/lib/postgresql/data/pgdata
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
  volumeClaimTemplates:
  - metadata:
      name: postgres-storage
    spec:
      storageClassName:  px-csi-db
      accessModes: [ "ReadWriteOnce" ]
      resources:
        requests:
          storage: 5Gi
  • Write data into the posgres
#!/bin/bash

# Configuration
POD_NAME="postgres-0"
DB_USER="postgres"

echo "--- Starting Database Setup on $POD_NAME ---"

# Combine all steps into a single SQL block for efficiency
SQL_COMMANDS=$(cat <<EOF
-- Step 1: Create Table
CREATE TABLE IF NOT EXISTS users (
    id SERIAL PRIMARY KEY,
    username TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Step 2: Write Data
INSERT INTO users (username)
SELECT 'user_' || seq
FROM generate_series(1, 100) AS seq;

-- Step 3: Force Hard Save (Checkpoint)
CHECKPOINT;
EOF
)

# Execute the SQL via oc exec
echo "Executing SQL commands..."
echo "$SQL_COMMANDS" | oc exec -i $POD_NAME -- psql -U $DB_USER

if [ $? -eq 0 ]; then
    echo "--- Success: Table created, data inserted, and disk synced. ---"
else
    echo "--- Error: Something went wrong during execution. ---"
fi
