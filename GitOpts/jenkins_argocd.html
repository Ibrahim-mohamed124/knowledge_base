CICD with GitOps

  • Usually we have two repos one for source code and another for IaC files
  • The GitOps operator that runs on a cluster will pull the IaC files from the Iac repo and applies them
  • A push or merge event on the application source code repo will create an image and update the IaC files in the another repo with the new image tag created by the CI in the a feature branch  and opens new PR to the main branch. 
  • A person checks the PR and  accepted it to the main branch
  • The GitOps operator refreshes the application and apply the changes
