Argo Events

  • Event driven workflow automation framework for  kubernetes and others
    • Event Sources: produce the event (webhooks, schedules, AWS notifies [s3, sqn,sns], and others)
    • EventSource object: consumes these events and transforms them into one standard format (CloudEvents format)
    • EventBus: central transport layer for events
      • Decouples the event sources from the event consumers by receiving the events as a subscriber from the EventSource and handling them independently 
    • Sensor:  Performs logic to handle the events from the Bus and trigger definitions when the event dependencies are met. 
    • Triggers: k8s jobs, workflow, notifier or any thing that triggered by an event  

Installation

kubectl create namespace argo-events
kubectl apply -f https://raw.githubusercontent.com/argoproj/argo-events/stable/manifests/install.yaml
