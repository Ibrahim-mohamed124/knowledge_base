Argo Events

  • Event driven workflow automation framework for  kubernetes and others
    • Event Sources: produce the event (webhooks, schedules, AWS notifies [s3, sqn,sns], and others)
    • EventSource object: consumes these events and transforms them into one standard format (CloudEvents format)
    • EventBus: central transport layer for events
      • Decouples the event sources from the event consumers by receiving the events as a subscriber from the EventSource and handling them independently 
    • Sensor:  Performs logic to handle the events from the Bus and trigger definitions when the event dependencies are met. 
    • Triggers: k8s jobs, workflow, notifier or any thing that triggered by an event

Event Bus

  • Central system for delivering the events
  • EventSource published the events to the Event Bus and the Sensor subscribes to the Event Bus depending on the4 NATS framwork

NATS is a lightweight, high-performance messaging system that supports pub/sub communication. It is designed for modern distributed systems like microservices and offers features such as simplicity, speed, and scalability. 

  • EventBus CRD creates a NATS cluster or connect to existing one.
apiVersion: argoproj.io/v1alpha1
kind: EventBus
metadata:
  name: default
spec:
  nats:
    native:
      replicas: 3 # optional, defaults to 3, and requires minimal 3
      auth: token # optional, defaults to none
      persistence: # optional
        storageClassName: standard
        accessMode: ReadWriteOnce
        volumeSize: 10Gi
