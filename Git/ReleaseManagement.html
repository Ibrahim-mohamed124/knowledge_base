Release Management

  • Release management is the process of planning, coordinating, and controlling how software changes move from development into production safely and predictably. 
    • In this process you have to balance speed with stability!!!

Release Types 

1. Scheduled Releases

  • Fixed release windows (e.g., bi-weekly, monthly)
  • Common in enterprises and regulated environments

2. On-Demand Releases

  • Released when features are ready
  • Enabled by CI/CD and automation

3. Emergency / Hotfix Releases

  • Minimal scope
  • Bypass some gates but still auditable

Release Gates (Quality Controls):

Common gates:

  • Automated tests
  • Code quality (SonarQube)
  • Security scans
  • Manual approval (CAB, product owner)
