What is Agile and DevOps?

  • Agile is a mindset not a framework with many values:
    • Agile Manifesto
    • Main Aim: optimization of fast feedback and adaptability
  • Agile optimizes the development and DevOps optimizes the delivery
  • DevOps is an extension of Agile to build, test, release, deploy, operate, monitor the code after it is done
  • Concepts of Agile:
    • iterative delivery
    • incremental value
    • continuous feedback
    • Cross functionality
    • fail fast > detect problems early (no technical dept)
  • Concepts of DevOps:
    • Automation > no silos
    • Instant feedback > no technical dept.
  • The three ways:

  1. Flow

  • Fast, smooth flow from code to production
  • Small batch sizes
  • Limit work in progress (WIP)

   2. Feedback

  • Fast feedback from:
  • CI tests
  • Monitoring
  • Logging
  • Alerts
  • Catch issues early

  3. Continuous Learning & Experimentation

  • Blameless postmortems
  • Continuous improvement
  • Automation over heroics

Agile Delivery Models

  • Agile delivery models describe how work is planned, developed, released (release strategy), and operated in an Agile environment.
    • Scrum: Fixed-length sprints (usually 2 weeks)
      • CI/CD must support frequent, sprint-based releases
      • Automation is critical to avoid sprint-end bottlenecks
    • Kanban: Continuous flow, no fixed sprints
      • WIP (Work In Progress) limits
      • Work visualized on a board
      • Aligns naturally with continuous delivery
      • Encourages small, frequent deployments
    • ScrumBan (Hybrid Model):  
      • Sprints exist
      • WIP limits applied
    • Lean Software Development
